Documentación de la API

Autenticación

Todas las solicitudes deben incluir el header X-API-Key:

X-API-Key: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

También se acepta como parámetro GET: ?api_key=...

Variables disponibles en plantillas

Al crear una plantilla, el cuerpo usa {% verbatim %}{{1}}{{2}}...{% endverbatim %} como marcadores de posición. Al configurar una campaña, mapeás cada posición a un campo del contacto usando estos nombres:

Nombre de variableQué traeAplica a
nombre_completoNombre completo del lead/clienteLeads y Clientes
emailEmailLeads y Clientes
planPlan de interés (Lead) o plan contratado (Cliente)Leads y Clientes
localidadLocalidadLeads y Clientes
provinciaProvinciaLeads y Clientes
telefonoTeléfono (formato +54...)Leads y Clientes

Campos personalizados (datos_extra): en la campaña elegís tipo "Campo extra" y escribís el slug del campo (el nombre en minúsculas, sin espacios, ej: empresa, cobertura_solicitada).

Valor fijo: texto que va igual para todos (ej: nombre de la empresa, fecha de un evento).

POST {{ base_url }}/api/v1/leads/ — Crear / actualizar lead

Crea un lead nuevo. Si el teléfono ya existe, actualiza los datos vacíos.

Request (JSON)
{
  "nombre_completo": "Juan Pérez",        // requerido
  "telefono": "1123456789",               // requerido
  "codigo_pais": "54",                    // opcional, default: 54
  "email": "juan@example.com",
  "dni": "35123456",
  "localidad": "Buenos Aires",
  "provincia": "Buenos Aires",
  "plan": "Individual",                   // nombre exacto del plan
  "notas": "Interesado por Google Ads",
  "origen": "web",                        // web|campaña|referido|llamada|whatsapp
  "datos_extra": { "utm_source": "google" }
}
Respuestas
201{"status":"created","lead_id":42,"telefono":"+541123456789"}
200{"status":"updated","lead_id":42,"telefono":"+541123456789"}
422{"error":"Teléfono inválido","code":"validation_error"}
401{"error":"API key inválida","code":"unauthorized"}
POST {{ base_url }}/api/v1/leads/actualizar/ — Actualizar campos de un lead existente

Busca el lead por teléfono y actualiza los campos que enviés. Diseñado para n8n y bots conversacionales.

Cualquier campo no reconocido se guarda automáticamente en datos_extra.

Request (JSON)
{
  "telefono": "+541123456789",            // requerido — identifica al lead

  // Campos estándar (todos opcionales, solo se actualizan los que enviés):
  "nombre_completo": "Juan Pérez",
  "email": "juan@example.com",
  "dni": "35123456",
  "localidad": "Buenos Aires",
  "provincia": "Buenos Aires",
  "notas": "Tiene 3 hijos",
  "plan": "Familiar",                     // nombre del plan
  "estado": "interesado",                 // nuevo|contactado|interesado|doc_pendiente|en_revision|afiliado|perdido
  "prioridad": "alta",                    // alta|media|baja

  // Campos extra (se mezclan con datos_extra existentes):
  "datos_extra": { "empresa": "Acme SA" },

  // También podés enviar campos extra directamente (van a datos_extra):
  "cobertura_solicitada": "completa",
  "cantidad_hijos": "2"
}
Respuestas
200{"status":"updated","lead_id":42,"telefono":"...","campos_actualizados":["email","datos_extra"]}
404{"error":"lead_not_found","telefono":"..."}
401{"error":"unauthorized"}

POST {{ base_url }}/api/v1/webhook/{source}/ — Webhook genérico

Igual que POST /leads/ pero el origen se toma del parámetro {source} de la URL.

POST {{ base_url }}/api/v1/webhook/landing-google/
# Lead creado con origen = landing-google en datos_extra
Integración con n8n — flujo conversacional por WhatsApp

Patrón recomendado para un bot de n8n que capta datos del contacto durante la conversación:

1. Crear el lead cuando llega el primer mensaje
POST {{ base_url }}/api/v1/leads/
{
  "nombre_completo": "{% verbatim %}{{$json.pushName}}{% endverbatim %}",     // nombre de WhatsApp
  "telefono":        "{% verbatim %}{{$json.from}}{% endverbatim %}",          // número del remitente
  "origen":          "whatsapp"
}
// Respuesta: {"status":"created","lead_id":123} o {"status":"updated",...}
// Guardá lead_id en la memoria del flujo n8n
2. El bot pregunta y el usuario responde — ir actualizando campos
POST {{ base_url }}/api/v1/leads/actualizar/
{
  "telefono": "{% verbatim %}{{$json.from}}{% endverbatim %}",
  "email":    "{% verbatim %}{{$json.respuesta_email}}{% endverbatim %}"
}

// O para un campo personalizado:
POST {{ base_url }}/api/v1/leads/actualizar/
{
  "telefono":            "{% verbatim %}{{$json.from}}{% endverbatim %}",
  "cobertura_solicitada":"{% verbatim %}{{$json.respuesta_cobertura}}{% endverbatim %}"
}
3. Cambiar el estado del lead según avance el flujo
POST {{ base_url }}/api/v1/leads/actualizar/
{
  "telefono": "{% verbatim %}{{$json.from}}{% endverbatim %}",
  "estado":   "interesado",
  "prioridad": "alta"
}
Campos de estado disponibles
nuevo contactado interesado doc_pendiente en_revision afiliado perdido
Número del remitente en Evolution API
// El webhook de Evolution API trae el número así:
"remoteJid": "5491112345678@s.whatsapp.net"

// Extraé el número con una función n8n:
{% verbatim %}{{$json.data.key.remoteJid.replace('@s.whatsapp.net','').replace('@c.us','')}}{% endverbatim %}
// Resultado: "5491112345678" — la API del CRM acepta con o sin "+"
Ejemplo completo con curl
# 1. Crear lead
curl -X POST {{ base_url }}/api/v1/leads/ \
  -H "Content-Type: application/json" \
  -H "X-API-Key: TU-API-KEY" \
  -d '{"nombre_completo":"María García","telefono":"1145678901","origen":"whatsapp"}'

# 2. Actualizar email cuando lo proporciona
curl -X POST {{ base_url }}/api/v1/leads/actualizar/ \
  -H "Content-Type: application/json" \
  -H "X-API-Key: TU-API-KEY" \
  -d '{"telefono":"+541145678901","email":"maria@gmail.com","estado":"interesado"}'

# 3. Guardar campo personalizado
curl -X POST {{ base_url }}/api/v1/leads/actualizar/ \
  -H "Content-Type: application/json" \
  -H "X-API-Key: TU-API-KEY" \
  -d '{"telefono":"+541145678901","cobertura_solicitada":"Plan Familiar","cantidad_hijos":"2"}'
